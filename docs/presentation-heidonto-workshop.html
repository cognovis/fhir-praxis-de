The Problem

Today: Every PMS Is an Island

  • Proprietary data models (GDT, BDT, xDT in Germany)
  • No standardized billing data model
  • No unified format for fee schedules (EBM/GOAe/BEMA/GOZ)
  • AI agents need structured data — it doesn't exist

Consequences

  • Every PMS switch = data loss
  • Every integration = custom build
  • AI billing assistants need a data model that doesn't exist
  • Internationally: zero comparability

Why Billing Needs a Data Model

Without a Standard

  • Every PMS stores fee schedules differently
  • Billing rules live in proprietary code
  • AI agents must be retrained per PMS
  • No verifiability: «Is this bill correct?»

With a FHIR Standard

  • ChargeItemDefinition for every fee schedule code
  • Extensions for points, multipliers, exclusions
  • Claim/ClaimResponse for prior authorizations
  • AI agent works PMS-independently

Odontathon 2026: Winner «Best Business Concept»

  • AI billing assistant built on FHIR profiles
  • Rules encoded in FHIR ChargeItemDefinition + CDS Hooks
  • Works for EBM, GOAe, BEMA, GOZ — same architecture

International Transferability

Analysis: Of 219 artifacts, ~30% are not Germany-specific.

Artifact Scope International Relevance
AI Provenance (8 extensions) EU EU AI Act applies EU-wide from 2026. No other IG has this.
CareTeam + Practitioner Roles International Team structure is universal. Role CodeSystem is adaptable.
Questionnaire / Intake Forms International SDC-compatible. Form templates are PMS-independent.
Appointment Mode DACH Telehealth modes. R5 only has virtualService (Zoom link), not the mode itself.
CAVE Warning Types DACH Standardized across PMS systems. AT/CH use similar codes.
Diagnosis Laterality (L/R/B) International Clinically universal. No IG has profiled this yet.
Patient Consent EU GDPR structure is transferable. Terminology is adaptable.

For HeyDonto / Dental internationally: The billing architecture (ChargeItemDefinition + Extensions) is transferable — only the tariff codes change.
